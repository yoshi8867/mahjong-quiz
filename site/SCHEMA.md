# SCHEMA — 마작패 표기법 + 문제 데이터 스키마

> **1~7절**: 패 표기법과 파서 동작 규약(M1 확정).
> **8~9절**: 문제 데이터 스키마와 봇 생성 규칙(M2 완성). 문제 생성 봇은
> 이 규약대로 손패 문자열과 문제 객체를 만들면 됩니다.

## 1. 기본 표기

숫자를 먼저 쓰고 **수트 문자**를 뒤에 붙입니다. 같은 수트의 연속 숫자는 묶어 쓸 수 있습니다.

```
123m456p789s11z
```

| 수트 문자 | 의미 | 숫자 범위 |
|-----------|------|-----------|
| `m` | 만수(萬子) | 1~9 |
| `p` | 통수(筒子) | 1~9 |
| `s` | 삭수(索子) | 1~9 |
| `z` | 자패(字牌) | 1~7 |

- `"123m456p789s11z"` → 1만 2만 3만, 4통 5통 6통, 7삭 8삭 9삭, 동 동 (11장)
- 수트 문자는 바로 앞에 나온 숫자 묶음 전체에 적용됩니다.

## 2. 적도라 (빨간 5)

수패에서 숫자 `0` 은 **적도라(빨간 5)** 를 뜻합니다. 랭크는 5로 정규화되고, 적도라 플래그가 붙습니다.

- `0m` = 빨간 5만, `0p` = 빨간 5통, `0s` = 빨간 5삭
- 파싱 결과: `{ suit: "m", num: 5, red: true }`
- 일반 `5m` 과 숫자(`num`)는 같지만 `red` 플래그로 구분됩니다.
- 자패에는 `0` 이 없습니다 (`0z` 는 오류).

## 3. 자패 번호 ↔ 유니코드 매핑 (주의)

z표기 번호와 유니코드 코드포인트 순서가 **다릅니다**. 삼원패(백/발/중)에서 어긋나므로 매핑 테이블로 처리합니다.

| z표기 | 패 | 유니코드 |
|-------|----|----------|
| `1z` | 동(東) | U+1F000 |
| `2z` | 남(南) | U+1F001 |
| `3z` | 서(西) | U+1F002 |
| `4z` | 북(北) | U+1F003 |
| `5z` | 백(白) | **U+1F006** |
| `6z` | 발(發) | **U+1F005** |
| `7z` | 중(中) | **U+1F004** |

수패 유니코드: 만수 1~9 = U+1F007~U+1F00F, 삭수 1~9 = U+1F010~U+1F018, 통수 1~9 = U+1F019~U+1F021.

## 4. 파서 API

`tiles.js` 는 `window.MahjongTiles` 로 노출됩니다 (테스트 편의용 `window.parseHand` alias 포함).

```js
window.MahjongTiles.parseHand("123m456p789s11z");
// → [{suit:"m",num:1,red:false}, ... , {suit:"z",num:1,red:false}]  (길이 11)
```

각 타일 객체: `{ suit: "m"|"p"|"s"|"z", num: 1~9, red: boolean }`

- `parseTile(token)` — 단일 토큰(`"5m"`, `"0p"`, `"7z"`) 을 타일 1개로. 정확히 1개가 아니면 throw.
- `tileCodePoint(tile)` / `tileGlyph(tile)` — 유니코드 글리프.
- `renderTile(tile, {mode, rotated})` — DOM 요소. `mode`: `"unicode"` | `"css"`.

## 5. 유효성 / 방어 동작 규약

| 입력 | 동작 |
|------|------|
| `""` (빈 문자열) | **`[]` 반환** (예외 아님) |
| `null` / `undefined` | `Error` throw |
| `"8z"`, `"9z"`, `"0z"` | `Error` throw (자패 1~7 범위 밖) |
| `"123"` (수트 접미어 없음) | `Error` throw |
| `"m123"` (숫자 앞 수트) | `Error` throw |
| `"12x3m"` (미지 수트 `x`) | `Error` throw |
| `"1.5m"`, `"!!"` (부호/구두점) | `Error` throw |

**원칙**: 유효하지 않은 입력은 **잘못된 타일을 만들지 않고** `Error` 를 던집니다.
빈 문자열만 빈 배열로 취급합니다. 호출부(쇼케이스/퀴즈 렌더)는 `try/catch` 로
감싸 예외가 페이지 로드를 중단시키지 않도록 합니다.

## 6. 스프라이트 시트 사양 (assets/tiles.png)

모바일에서 유니코드 마작 글리프가 깨지는(두부) 문제 때문에 최종 렌더는 미리 구운
**스프라이트 PNG 한 장**을 셀 단위로 잘라 쓴다. 쇼케이스가 이 PNG를 굽는 제너레이터를
겸한다(캔버스 미리보기 = 저장될 파일).

### 그리드 (9열 × 5행)

| 행 | col0 | col1 | col2 | col3 | col4 | col5 | col6 | col7 | col8 |
|----|------|------|------|------|------|------|------|------|------|
| 0 | 1m | 2m | 3m | 4m | 5m | 6m | 7m | 8m | 9m |
| 1 | 1p | 2p | 3p | 4p | 5p | 6p | 7p | 8p | 9p |
| 2 | 1s | 2s | 3s | 4s | 5s | 6s | 7s | 8s | 9s |
| 3 | 1z(동) | 2z(남) | 3z(서) | 4z(북) | **5z(백=빈몸체)** | 6z(발) | 7z(중) | (빈칸) | (빈칸) |
| 4 | 0m(적) | 0p(적) | 0s(적) | 빈타일 | 뒷면 | — | — | — | — |

- **셀 크기 (고정)**: 반응형 브레이크포인트와 무관하게 **166×248px** 상수로 굽는다.
  패 몸체가 셀에 상하좌우 **여백 없이 꽉 차며**, 셀 간 간격·그림자·라운드는 CSS(`.tile--sprite`)가
  담당한다. 창 너비에 따라 시트 크기가 달라지지 않는다.
- **시트 크기**: `166×9 = 1494` × `248×5 = 1240` → **1494×1240px** (9열×5행).
- **셀 좌표 공식**: `x = col × 166`, `y = row × 248` (col 0~8, row 0~4).
- **몸체**: 라운드 사각형 아이보리(`--color-tile`). **그림자·테두리 선은 굽지 않음**
  (그림자·라운드는 CSS `.tile--sprite`가 담당). 라운드 모서리 밖은 투명.
- **내용물**(DOM 커스텀 타일과 동일 규칙):
  - 만수: 한자(`一萬`~`九萬`, 숫자 위·`萬` 아래 세로 배치). 적도라 `0m`=`五萬` 빨강.
  - 통·삭: 유니코드 마작 글리프. `--glyph-scale`(타일 높이 대비)만큼 확대하고 몸체
    라운드 경로로 `ctx.clip()` → 글리프 자체 테두리가 잘려나감(= `overflow:hidden` 재현).
    적도라 `0p/0s` 글리프는 빨강.
  - 자패: 한자. **中 빨강, 發 녹색**, 동남서북은 먹색. 백(5z)은 내용 없는 빈 몸체.
  - 뒷면: 어두운 단색(`--color-felt-dark`).
- 캔버스는 선택된 `--font-hanzi` 를 그대로 쓰고 `textAlign='center'`, `textBaseline='middle'`.

### 좌표 매핑 (렌더 시)

`MahjongTiles.spriteIndex(tile) → { col, row }`:
- 적도라(`red:true`) → `row 4`, `col = {m:0, p:1, s:2}`
- 그 외: `m→row0, p→row1, s→row2, z→row3`, `col = num - 1`

CSS 스프라이트 모드는 퍼센트 좌표로 셀을 자른다(셀 종횡비 = 타일 종횡비라 반응형 유지):

```
background-size: 900% 500%;                 /* 9열 × 5행 */
background-position: (col/8*100)% (row/4*100)%;
```

## 7. 레이아웃 프리미티브 (렌더 헬퍼)

- `renderHand(handStr, {mode, tsumo})` — 손패 가로 나열, `tsumo` 지정 시 간격 분리.
- `renderMeld(handStr, {mode, rotatedIndex})` — 부른 패 세트, 지정 위치 1장 90도 회전.
- `renderDiscards(handStr, {mode})` — 버림패 강, 6장씩 줄바꿈(6열 그리드).
- 회전 타일은 눕힌 비율(`--tile-h` × `--tile-w`)만큼 자리를 차지해 겹치지 않습니다.

> **본편(퀴즈) 렌더 기본 모드는 `sprite`**입니다. 손패·도라·부른 패·보기 타일 모두
> `renderTile(tile, {mode:'sprite'})`(= `.tile--sprite` + inline `background-position`)로
> 그립니다. `unicode`/`css` 모드는 쇼케이스 비교용입니다.

## 8. 문제 데이터 스키마 (`js/questions.js`)

문제은행은 `window.MahjongQuestions`(배열)로 노출됩니다. 각 문제 객체:

```js
{
  id: "d001",                 // 문자열, 고유. 관례: 유형접두어(d/y/s/r) + 3자리
  type: "discard",            // "discard" | "yaku" | "score" | "rule"
  difficulty: 3,               // 1~5 정수
  prompt: "무엇을 버릴까?",     // 지문(비어있으면 안 됨)
  hand: "234m567m55p23p4678s", // 표준 패 표기. 개념 문제는 "" 허용(타일 미표시)
  draw: "4s",                  // (선택, discard 전용) 방금 쯔모한 패 1장. hand에 포함돼야 함
  dora: "2p",                  // (선택) 도라 표시패 1장 표기
  melds: ["p777z"],            // (선택) 부른 패 배열. 접두어 + 표기
  discards: "1p9m6s2p3m7z",    // (선택) 버림패 강 표기
  choices: ["2p", "4s", "6s", "5p"], // 길이 4 배열
  answer: 1,                   // 정답 index 0~3
  explanation: "…"             // 해설(비어있으면 안 됨)
}
```

### 필드 규칙

| 필드 | 필수 | 규칙 |
|------|------|------|
| `id` | ✔ | 문자열, 전체에서 고유 |
| `type` | ✔ | `discard`/`yaku`/`score`/`rule` 중 하나 |
| `difficulty` | ✔ | 1~5 정수 |
| `prompt` | ✔ | 공백만 있으면 안 됨 |
| `hand` | ✔ | 표준 표기. 파싱 가능해야 함(개념 문제는 `""`) |
| `draw` | 선택 | `discard` 전용. 1장 표기이며 **`hand`에 존재하는 패**여야 함 |
| `dora` | 선택 | 정확히 1장으로 파싱 |
| `melds` | 선택 | 각 요소 = 접두어(`p`퐁/`c`치/`k`깡) + 표기. 표기부는 파싱 가능 |
| `discards` | 선택 | 표준 표기(강). 파싱 가능 |
| `choices` | ✔ | 배열, 길이 정확히 4 |
| `answer` | ✔ | 0~3 정수 |
| `explanation` | ✔ | 공백만 있으면 안 됨 |

### `choices` 렌더 규칙 (패 표기 ↔ 텍스트 자동 판별)

`MahjongEngine.parseChoiceTiles(str)`가 보기 문자열을 판별한다:

- `parseHand(str)`가 예외 없이 **길이>0** 배열을 반환하면 → **타일(스프라이트)** 로 렌더.
  예: `"9s"`, `"3z"`, `"55s"`.
- 예외가 나거나 빈 배열이면 → **텍스트 그대로** 렌더.
  예: `"탕야오"`, `"1000점"`, `"2000/2000/4000"`.

즉 점수·역명·설명형 보기는 자연히 텍스트로, 순수 패 표기는 타일로 나온다. 봇은 별도 플래그
없이 문자열만 넣으면 된다(점수 보기에는 `"점"` 등 비패 문자를 포함시켜 텍스트로 유도).

### 유형별 작성 지침

- `discard` — 14장 `hand` + 최적 타패. **UI는 4지선다 버튼이 아니라 손패의 각 패를 직접 클릭**해 답한다. 따라서 `choices[answer]`(정답 index가 가리키는 보기)가 **정답 패 표기**가 되며(예 `"5s"`), `choices`의 나머지도 패 표기로 둔다. 판정은 클릭한 패의 num+suit를 `choices[answer]`와 비교(적도라 `0s`는 `5s`와 동등 취급). 같은 표기의 패가 손에 여러 장이면 어느 것을 클릭해도 정답. `draw`를 지정하면 그 1장을 손패에서 분리해 우측(쯔모 자리)에 간격을 두고 표시하며, 이 쯔모패도 클릭(쯔모기리) 가능하다. 필요 시 `dora`/`discards` 첨부.
- `yaku` — 완성/준완성 손패. `choices`는 역명(텍스트). 성립 역이 **하나만** 맞도록 오답 보기를 구성.
- `score` — `hand`는 보통 `""`. 부수/판수·쯔모/론·장가/자가 조건을 `prompt`에 명시, `choices`는 점수 텍스트.
- `rule` — 후리텐/울기/리치/창깡 등 판단. `choices`는 텍스트. 필요 시 `hand`·`discards`로 상황 예시.

## 9. 저장소 / 엔진 전역 노출

| 전역 | 내용 |
|------|------|
| `window.MahjongTiles` | 파서 + 렌더(M1) |
| `window.parseHand` | `parseHand` alias |
| `window.MahjongQuestions` | 문제 배열 |
| `window.MahjongEngine` | `filter` / `pickWeighted` / `parseChoiceTiles` |
| `window.MahjongStorage` | localStorage 래퍼(오답·통계) |

- **출제**: `MahjongEngine.filter({types, minDifficulty, maxDifficulty})` → 후보 배열.
  `pickWeighted(pool, {excludeId, stats})` → 가중 랜덤 1문제(직전 문제 회피, 유형별
  정답률 낮을수록 가중↑, 통계 없으면 균등).
- **필터 전달**: 홈 → 퀴즈는 **쿼리스트링** `quiz.html?types=rule&min=4&max=5`.
- **저장 키**: 오답 `mjq.wrong`(배열), 통계 `mjq.stats`(`{type:{}, diff:{}}`). 접근 불가·깨진
  JSON·타입 불일치 시 `try/catch`로 삼키고 기본값으로 복구한다.
