# PLAN — 리치마작 4지선다 학습 사이트

## 1. 티어와 이유

**T1** (순수 HTML/CSS/JS, 멀티 페이지, 빌드 없음). 상태 저장은 localStorage, 데이터는 정적 JS 파일이면 충분하고, 타일 렌더링은 CSS + 소량의 DOM 생성 코드로 구현 가능하다. 반응형 UI 갱신(선택 → 정오 판정 → 해설 표시)도 바닐라 JS 이벤트 핸들링 수준이라 Alpine.js조차 필수는 아니므로 CDN 의존성 없이 진행한다(구현 중 상태 바인딩이 번거로우면 Alpine.js 한 줄 추가 허용). T2가 필요한 근거가 없다.

## 2. 파일 구조

```
site/
├── index.html          # 홈/설정: 유형·난이도 선택, 퀴즈 시작, 통계 요약
├── quiz.html           # 퀴즈: 문제 + 4지선다 + 정오/해설
├── review.html         # 오답 노트: 목록 + 재풀이
├── stats.html          # 통계: 유형별/난이도별 정답률
├── showcase.html       # 타일 쇼케이스 데모 (개발용, 최종 산출물에도 유지)
├── styles.css          # 전체 스타일 (CSS 커스텀 프로퍼티 토큰 포함)
├── js/
│   ├── tiles.js        # 타일 표기 파서 + 타일/레이아웃 렌더링 컴포넌트
│   ├── questions.js    # 문제은행 데이터 (샘플 30문제) + 스키마 주석
│   ├── storage.js      # localStorage 래퍼 (오답, 유형별 통계)
│   ├── engine.js       # 적응형 가중 랜덤 출제 + 정오 판정 로직
│   └── app.js          # 각 페이지 초기화/이벤트 바인딩
└── SCHEMA.md           # 문제 데이터 스키마 + 패 표기법 문서 (봇 생성용 레퍼런스)
```

## 3. 디자인 방향

- 색: 마작탁 느낌의 딥그린(`--color-felt: #1e5e3f`) 배경 + 아이보리 타일(`--color-tile: #f6f1e3`), 포인트는 적도라 레드(`--color-accent: #c0392b`). 정답 그린/오답 레드 피드백 색 토큰 별도.
- 폰트: 시스템 산세리프 스택(한국어 UI), 타일 내 한자/숫자는 serif 계열 폴백 지정. 유니코드 타일 글리프용 폰트 스택 별도 토큰.
- 레이아웃: 최대 폭 720px 중앙 컬럼, 카드형 문제 패널. 모바일 우선(타일 크기 `--tile-w`/`--tile-h` 토큰으로 일괄 스케일).
- 모든 색·간격·타일 치수는 `:root`의 CSS 커스텀 프로퍼티 토큰으로 정의하고 하드코딩 금지 — 사용자 검토 게이트에서 디자인 수정이 반복될 수 있으므로 토큰 중심으로 조정 가능하게.

## 4. 문제 데이터 스키마 (요지 — 상세는 site/SCHEMA.md에 문서화)

```js
{
  id: "d001",            // 유형접두어 + 일련번호
  type: "discard",       // "discard" | "yaku" | "score" | "rule"
  difficulty: 3,          // 1~5
  prompt: "동1국 남가, 무엇을 버릴까?",  // 지문
  hand: "123m456p78s11z5s",  // 표준 표기 (m=만수, p=통수, s=삭수, z=자패 1~7, 0=적5)
  dora: "3p",             // 도라 표시패 (선택)
  melds: ["p555m", "c123s"], // 부른 패 (선택; p=퐁, c=치, k=깡 + 표기)
  discards: "1z9m2p...",  // 버림패 강 (선택, rule/discard 문제용)
  choices: ["1z", "5s", "탕야오", "3900점"], // 보기 4개 (패 표기면 타일로 렌더링)
  answer: 1,              // 정답 index 0~3
  explanation: "…"        // 해설
}
```

- `tiles.js`의 파서가 `"123m456p789s11z"` 형식을 타일 배열로 변환. 보기 문자열도 패 표기로 인식되면 타일로 렌더링, 아니면 텍스트 그대로.
- 봇이 텍스트만으로 문제를 대량 생성할 수 있도록 SCHEMA.md에 표기법 전체(적도라 `0`, 부른 패 접두어, 회전 위치 규칙)와 유효성 규칙을 명시.

## 5. 마일스톤

### M1: 타일 렌더링 컴포넌트 + 쇼케이스 (★ 완료 후 사용자 검토 게이트)
**산출물**: `site/showcase.html`, `site/styles.css`, `site/js/tiles.js`, `site/SCHEMA.md`(표기법 부분)

- 표준 표기 파서 (`123m456p789s11z`, 적도라 `0m/0p/0s`, 자패 `1z~7z`)
- 타일 렌더링 두 방식: (a) 유니코드 글리프(U+1F000~U+1F021), (b) CSS 커스텀 타일(라운드 사각형 흰 몸체 + 그림자, 내부에 글리프 또는 한자/숫자)
- 90도 회전 타일 지원, 레이아웃 프리미티브: 손패(가로), 도라 표시패, 부른 패 세트(1장 회전), 버림패 강(6장×줄)

**수용 기준**:
1. showcase.html을 열면 전체 34종 타일이 유니코드 방식과 CSS 커스텀 방식 두 줄로 나란히 비교 표시된다.
2. 90도 회전된 타일이 포함된 퐁/치 세트와 6장씩 줄바꿈되는 버림패 강 샘플이 보이고, 회전 타일의 비율·정렬이 깨지지 않는다.
3. 손패 13장 + 쯔모 1장(간격 분리) + 도라 표시패 레이아웃 샘플이 표시된다.
4. 적도라(빨간 5)가 일반 5와 시각적으로 구분된다.

**→ M1 완료 후 사용자 검토를 받고, 피드백 반영(디자인 수정 반복 가능) 후 M2 진행.**

### M2: 퀴즈 코어 (문제은행 + 출제 + 정오/해설)
**산출물**: `site/index.html`, `site/quiz.html`, `site/js/questions.js`, `site/js/engine.js`, `site/js/storage.js`, `site/js/app.js`, `site/SCHEMA.md`(완성)

- 샘플 30문제(유형 4종 × 난이도 1~5 고르게), 스키마 문서 완성
- 홈에서 유형·난이도 범위 선택 → 퀴즈 시작, 가중 랜덤 출제(초기엔 균등, 통계 쌓이면 약한 유형 가중)

**수용 기준**:
1. index.html에서 유형/난이도 범위를 선택하고 시작하면 quiz.html에서 해당 조건의 문제가 타일 렌더링과 함께 표시된다.
2. 보기 4개 중 하나를 클릭하면 즉시 정답/오답이 색으로 표시되고 해설이 나타난다.
3. "다음 문제" 클릭 시 새 문제가 출제되고, 같은 문제가 연속으로 반복되지 않는다.
4. 오답 시 localStorage에 기록이 남는다(개발자 도구에서 확인 가능).

### M3: 오답 노트 + 통계 + 적응형 마감
**산출물**: `site/review.html`, `site/stats.html`, `engine.js`/`storage.js`/`app.js` 보강

**수용 기준**:
1. 틀린 문제가 review.html 목록에 나타나고, 재풀이에서 맞히면 목록에서 해소된다.
2. stats.html에서 유형별·난이도별 정답률이 표시되고, 홈에도 요약이 보인다.
3. 특정 유형의 정답률을 낮게 만들면(여러 번 틀리면) 이후 출제에서 그 유형이 체감상 더 자주 나온다.
4. 브라우저를 새로고침해도 오답·통계가 유지된다.
