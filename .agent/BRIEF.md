# BRIEF — 마작(리치마작) 학습 웹사이트

## 목적
리치마작 실력 향상을 위한 4지선다 문제 풀이 학습 사이트. 사용자의 약한 유형을 파악해 적응형으로 출제하고, 오답을 복습할 수 있게 한다.

## 대상 사용자
- 리치마작 규칙을 어느 정도 아는 학습자 (초~중급)
- 언어: 한국어 UI (마작 용어는 한국 마작계 관용 표기: 타패, 후리텐, 탕야오 등)

## 핵심 기능
1. **4지선다 퀴즈** — 문제 1개씩 표시, 보기 4개 중 선택, 즉시 정오 판정 + 해설 표시.
2. **오답 정리** — 틀린 문제를 자동 저장(localStorage), 오답 노트 페이지에서 다시 보기/재풀이 가능. 재풀이에서 맞히면 오답 목록에서 해소 처리.
3. **적응형 출제** — 유형별 정답률을 추적해 약한 유형이 더 자주 나오도록 가중 랜덤 출제. 통계 페이지에서 유형별/난이도별 정답률 확인 가능.
4. **난이도 필터** — 문제마다 난이도 1~5. 사용자가 난이도 범위를 골라 풀 수 있음.

## 문제 유형 (적응형 출제의 축, 4종 모두 포함)
- `discard` — 무엇을 버릴까(何切る): 손패 13~14장 + 도라 표시, 최적 타패 선택
- `yaku` — 역(役) 판정: 완성 손패를 보고 성립 역 고르기
- `score` — 점수 계산: 부수/판수 → 점수, 또는 손패 → 최종 점수
- `rule` — 룰/판단: 후리텐 여부, 울기 가능 여부, 리치 가능 여부 등

## 문제 데이터
- 초기 문제은행: **샘플 30문제** 내장 (유형×난이도 고르게 분포).
- **중요**: 사용자가 나중에 문제 생성 봇을 만들어 계속 추가할 예정. 따라서 문제 데이터는 별도 JS/JSON 파일로 분리하고, **스키마를 명확히 문서화**할 것 (id, type, difficulty 1~5, 문제 지문, 손패 표기, 보기 4개, 정답 index, 해설). 손패 표기는 표준 마작 표기(예: "123m456p789s11z" 식) 파서를 두어 봇이 텍스트로 문제를 생성하기 쉽게.

## ★ 마작패 시각 디자인 (최우선 품질 항목, 사용자 직접 검토 게이트 있음)
- 기본 접근: 유니코드 마작패 문자(U+1F000~U+1F021) 사용. 단, 플랫폼별 글리프 렌더링 편차가 크므로:
  - 대안/보완: CSS로 라운드 사각형 타일 배경(흰 몸체 + 그림자, 아이보리 뒷면 느낌)을 직접 그리고 그 안에 유니코드 글리프 또는 한자/숫자 텍스트를 배치하는 커스텀 타일 컴포넌트도 구현.
  - 두 방식 모두 구현해 비교 가능하게 하는 것이 이상적.
- **회전 표시 지원 필수**: 다른 사람이 버린 패(강 표시), 퐁/치로 가져온 패는 90도 회전(옆으로 눕힘)해 표시. 회전 시에도 타일 비율·정렬이 자연스러워야 함.
- 손패(가로 나열), 도라 표시패, 부른 패(퐁/치 세트, 1장 회전), 버림패(강, 6장씩 줄) 등의 레이아웃 프리미티브 필요.
- **마일스톤 구성 요구**: 타일 렌더링 컴포넌트를 독립된 초기 마일스톤으로 분리하고, 해당 마일스톤 완료 시 데모/쇼케이스 페이지(모든 패 + 회전 + 레이아웃 샘플)를 만들어 **사용자 검토를 받은 뒤** 다음 마일스톤 진행. 사용자 피드백에 따라 타일 디자인 수정 반복 가능성 있음.

## 페이지 구성 (제안)
- 홈/설정: 유형·난이도 선택, 퀴즈 시작, 통계 요약
- 퀴즈: 문제 표시(패 렌더링), 4지선다, 정오+해설
- 오답 노트: 틀린 문제 목록, 재풀이
- 통계: 유형별/난이도별 정답률
- 타일 쇼케이스(개발용 데모 페이지): 전체 패 렌더링 + 회전 + 레이아웃 검증용 — 최종 산출물에도 남겨둠

## 데이터 저장
- 서버 없음. 진행 상태·오답·통계 모두 localStorage.

## 티어 제약
기본 T1 (순수 HTML/CSS/JS + 필요 시 Alpine.js/htmx CDN). T2(Vite)는 T1 불가 사유를 정당화해야만 허용. 무거운 프레임워크·Tailwind·백엔드 금지.
